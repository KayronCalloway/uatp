## Executive Summary

The Unified Agent Trust Protocol (UATP) is a cryptographically verifiable trust architecture for intelligent systems. It transforms blind faith in AI into structured proof.

UATP began with a simple observation: most AI failures don't stem from a lack of intelligence—they stem from a lack of architecture. Today's systems don't need smarter models. They need auditable ones.

UATP provides a universal substrate for trust—scalable across assistants, agents, critical systems, and institutions. It does so through cryptographic capsules, refusal logging, reasoning transparency, and verifiable memory trails.

## Origin: From Siri to Systemic Accountability

UATP originated from a foundational failure: assistants that couldn't explain, undo, or justify their actions. That failure prompted four persistent questions:

1. Can I trust what this agent did?
2. Can I prove what the agent knew when it acted?
3. Can I audit how the output was generated?
4. Can I intervene before damage is done?

UATP 6.0 answers all four—across agents, industries, and jurisdictions.

## Capsule Architecture

At the core of UATP is the Capsule—a cryptographically sealed, inspectable unit of cognition, action, refusal, or co-authorship. Capsules form Capsule Chains, creating tamper-proof records of agent behavior.

### Capsule Types

| Type | Function |
| --- | --- |
| Refusal | Logs ethical or operational rejection of action |
| Introspective | Records internal reasoning, uncertainty, alternatives |
| Joint | Co-authored decisions, dual-signed (human + AI) |
| Meta | Reflects on capsules to detect drift or contradiction |
| Influence | Tracks persuasive or emotional impact on humans |
| Perspective | Declares epistemic, cultural, or ideological framing |
| Lifecycle | Logs startup, shutdown, and memory events |
| Embodied | Captures physical interaction (vision, sound, touch) |
| Ancestral Knowledge | Tags public-domain or foundational insight lineage |

Capsule Chains are linear by default but support recursive inspection and forking. When contradictions arise (e.g. Meta vs. Refusal), a Perspective Fork is initiated—preserving all branches and logging divergence transparently. No capsule deletes truth—only reframes it.
