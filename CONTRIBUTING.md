# Contributing to UATP

Thank you for your interest in contributing to UATP!

## How to Contribute

### Reporting Issues

- Use [GitHub Issues](https://github.com/KayronCalloway/uatp/issues)
- Include steps to reproduce
- Include your environment (OS, Python version)

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Run tests (`pytest`)
5. Commit with clear messages
6. Push and open a Pull Request

### Code Style

- We use `ruff` for linting and formatting
- Run `pre-commit install` to set up hooks
- Type hints are required for new code

### Security

If you discover a security vulnerability, please email Kayron@houseofcalloway.com instead of opening a public issue.

## Questions?

Open a [Discussion](https://github.com/KayronCalloway/uatp/discussions) or email Kayron@houseofcalloway.com.
